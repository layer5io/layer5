Tables are used to organize and display structured data in rows and columns. They make it easy for users to scan, compare, and interact with information. The Table component in Sistent is built on Material UI primitives and supports full theming.

<a id="Overview">
  <h2>Overview</h2>
</a>

The <strong>Table</strong> component organizes information into rows and columns, making it easy for users to read, compare, and analyze data. Use a Table when displaying multiple records with consistent attributes such as services, users, or deployments, when users need to compare values across rows or columns, or when data needs sorting, filtering, or pagination for large data sets.

<a id="Anatomy">
  <h2>Anatomy</h2>
</a>

<p>A Sistent Table is composed of the following sub-components:</p>

<ul>
  <li><strong>Table</strong> — Root container element.</li>
  <li><strong>TableHead</strong> — Contains the header row with column labels.</li>
  <li><strong>TableBody</strong> — Contains the data rows.</li>
  <li><strong>TableRow</strong> — A single row within the head or body.</li>
  <li><strong>TableCell</strong> — An individual cell, used for both header and data cells.</li>
  <li><strong>TableFooter</strong> — Optional footer area, typically used for pagination controls.</li>
  <li><strong>TablePagination</strong> — A helper component for adding page-size and navigation controls.</li>
  <li><strong>TableSortLabel</strong> — A helper component for rendering a sortable column header with an arrow icon.</li>
  <li><strong>TableContainer</strong> — A wrapper that enables horizontal scrolling on overflow.</li>
</ul>

<a id="Variants">
  <h2>Variants</h2>
</a>

<h3>Basic Table</h3>

The simplest form of a Table — static, read-only rows with a header.

<h3>Sortable Table</h3>

Columns whose headers include <code>TableSortLabel</code> allow users to reorder rows by ascending or descending value.

<h3>Selectable Rows Table</h3>

Rows include a leading <code>Checkbox</code> component so users can select one or more records for bulk actions.

<h3>Paginated Table</h3>

When a data set is large, <code>TablePagination</code> is added to the <code>TableFooter</code> to control which slice of data is displayed.

<a id="Accessibility">
  <h2>Accessibility</h2>
</a>

<ul>
  <li>Table headers should always be present with meaningful column labels.</li>
  <li>Provide a meaningful <code>aria-label</code> on the Table root when its purpose is not obvious from surrounding context.</li>
  <li>Sortable columns expose <code>aria-sort</code> automatically via <code>TableSortLabel</code>.</li>
  <li>Ensure all interactive elements inside cells are keyboard-focusable.</li>
</ul>
