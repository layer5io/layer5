---
title: Table Guidance
component: table
description: A table displays structured data in rows and columns, enabling users to scan, compare, and interact with information efficiently.
---

For proper application, the Table component can be used for different purposes to enable easier and consistent display of structured data across digital experiences.

<a id="Do">
  <h2>Do</h2>
</a>

<ul>
  <li>Use tables for homogeneous data sets with three or more columns.</li>
  <li>Add a caption or <code>aria-label</code> for assistive technologies.</li>
  <li>Use <code>size="small"</code> for dense data-heavy views like log tables.</li>
  <li>Wrap the table in <code>TableContainer</code> to enable horizontal scrolling on narrow viewports.</li>
  <li>Use <code>TablePagination</code> when row count exceeds 25 with page sizes <code>[10, 25, 50]</code>.</li>
  <li>Place inline row actions in the last column, right-aligned, using <code>IconButton</code> with a descriptive <code>aria-label</code>.</li>
  <li>Always confirm destructive actions like delete before execution.</li>
</ul>

<a id="Dont">
  <h2>Don&#39;t</h2>
</a>

<ul>
  <li>Use tables purely for layout alignment of unrelated content.</li>
  <li>Mix unrelated data types across rows.</li>
  <li>Omit column headers — every table must have a <code>TableHead</code>.</li>
  <li>Hard-code hex colour values — always use Sistent theme tokens.</li>
  <li>Rely on colour alone to convey row status; supplement with text or icons.</li>
</ul>

<a id="Row Density">
  <h2>Row Density</h2>
</a>

<p>Sistent exposes a <code>size</code> prop on the Table component:</p>

<ul>
  <li>Use <strong><code>size="medium"</code></strong> (default) for general-purpose tables with comfortable vertical spacing.</li>
  <li>Use <strong><code>size="small"</code></strong> for dense data-heavy views where vertical space is at a premium, such as log tables or comparison matrices.</li>
</ul>

<a id="Row Selection">
  <h2>Row Selection</h2>
</a>

<ul>
  <li>Place a <code>Checkbox</code> in the first cell of each row and in the header row for select-all behaviour.</li>
  <li>Update the row background to <code>theme.palette.action.selected</code> on selection.</li>
  <li>Show a selected-row count in a toolbar above the table.</li>
  <li>A single click on the checkbox should toggle selection.</li>
</ul>

<a id="Empty and Loading States">
  <h2>Empty and Loading States</h2>
</a>

<ul>
  <li><strong>Empty:</strong> Display a single full-width <code>TableCell</code> spanning all columns with a brief message and call-to-action.</li>
  <li><strong>Loading:</strong> Replace body rows with skeleton rows or a centered <code>CircularProgress</code> inside a spanning cell.</li>
</ul>
