<h3>Introduction</h3>

<p>In this tutorial, we explore how to use <strong>Kubernetes ConfigMaps and Secrets</strong> with Meshery Playground. ConfigMaps allow you to decouple configuration from container images, while Secrets store sensitive data such as passwords and API keys securely.</p>

<h3>Prerequisites</h3>

<ul>
    <li>Basic understanding of Kubernetes Pods and Deployments.</li>
    <li>Access to the <em>Meshery Playground</em>. Sign up at <a href="https://play.meshery.io/">Meshery Playground</a> if needed.</li>
</ul>
<h3>Lab Scenario</h3>

<p>Create a ConfigMap and a Secret, then mount them into a Pod using Meshery's visual design canvas.</p>

<h3>Objective</h3>

<p>Learn how to create <strong>ConfigMaps</strong> and <strong>Secrets</strong> and attach them to Pods using Meshery Playground.</p>

<h3>Steps</h3>

<h4>Access Meshery Playground</h4>

<ol>
    <li>Log in to <a href="https://play.meshery.io">Meshery Playground</a> and click <strong>Explore</strong> to open <em>Kanvas</em>.</li>
    <li>Rename your design (e.g., <code>configmap-secrets-lab</code>).</li>
</ol>
<h4>Create a ConfigMap</h4>

<ol>
    <li>In the <strong>Components</strong> tab, search for <strong>ConfigMap</strong> and drag it onto the canvas.</li>
    <li>Click the ConfigMap component to open its configuration.</li>
    <li>Under <strong>data</strong>, add a key-value pair, e.g.:</li>
</ol>   - Key: <code>APP_ENV</code>
<p>- Value: <code>production</code></p>
<ol>
    <li>Give it a name (e.g., <code>app-config</code>) and a label.</li>
</ol>
<h4>Create a Secret</h4>

<ol>
    <li>Search for <strong>Secret</strong> in Components and drag it onto the canvas.</li>
    <li>Click the Secret component and add base64-encoded data under <strong>data</strong>, e.g.:</li>
</ol>   - Key: <code>DB_PASSWORD</code>
<p>- Value: <code>cGFzc3dvcmQ=</code> (base64 for "password")</p>
<ol>
    <li>Name it (e.g., <code>app-secret</code>).</li>
</ol>
<h4>Create a Pod and Mount ConfigMap and Secret</h4>

<ol>
    <li>Drag a <strong>Pod</strong> component onto the canvas.</li>
    <li>Add a container (<code>nginx:latest</code>) as covered in the Pods tutorial.</li>
    <li>In the container configuration, scroll to <strong>envFrom</strong> to reference the ConfigMap:</li>
</ol>    - Add a <code>configMapRef</code> pointing to <code>app-config</code>.
<ol>
    <li>Add another entry under <strong>envFrom</strong> referencing the Secret:</li>
</ol>    - Add a <code>secretRef</code> pointing to <code>app-secret</code>.
<ol>
    <li>Add a unique label to the pod for filtering.</li>
</ol>
<h4>Validate and Deploy</h4>

<ol>
    <li>Click <strong>Validate</strong> and resolve any errors.</li>
    <li>Click <strong>Deploy</strong> to apply the design.</li>
</ol>
<h4>Verify in Visualize</h4>

<ol>
    <li>Switch to <strong>Visualize</strong> and filter by your pod's label.</li>
    <li>Click the pod and examine the <strong>Details</strong> tab to confirm the configuration is applied.</li>
</ol>
<h4>Clean Up</h4>

<ol>
    <li>Use <strong>Undeploy</strong> to remove all resources.</li>
</ol>
<h3>Conclusion</h3>

<p>You've successfully created Kubernetes ConfigMaps and Secrets and mounted them into a Pod using Meshery Playground. This approach enables clean separation of configuration from application code.</p>
